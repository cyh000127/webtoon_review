# 웹툰 기록 페이지 작업 정의

## 목표

개인 웹툰 감상 기록을 정리하는 TypeScript 기반 웹 페이지를 만든다. 불법 공유나 외부 감상 기능이 아니라, 사용자가 이미 읽었거나 읽는 중인 웹툰을 개인적으로 분류하는 기록용 페이지다.

데이터베이스 없이 JSON 파일과 로컬 브라우저 저장소만 사용한다. 작품 데이터는 저장소 안의 정적 파일로 관리하고, 독자 평점은 같은 브라우저의 `localStorage`에만 저장한다.

## 핵심 화면

참고 이미지처럼 어두운 배경의 웹툰 목록형 UI를 만든다.

- 상단에는 큰 상태 탭을 둔다.
  - `연재중`
  - `완결`
- 그 아래에는 독서 상태 세부 탭을 둔다.
  - `전체`
  - `읽는 중`
  - `완주`
  - `중도 포기`
- 웹툰 목록은 카드 보기와 목록 보기로 전환할 수 있다.
- 카드와 목록에는 표지, 플랫폼, 장르, 제목, 작가, 진행 상태, 작품 소개 요약을 보여준다.
- 검색, 플랫폼 필터, 장르 필터, 정렬 옵션을 제공해 기록이 많아져도 찾기 쉽게 한다.
- 통계 대시보드에는 전체 작품 수, 완주율, 최근 업데이트, 감상 상태 분포, 장르 TOP 5를 그래프 형태로 보여준다.

## 상태 모델

웹툰 하나는 두 가지 축으로 분류한다.

1. 작품 연재 상태
   - `ongoing`: 연재
   - `completed`: 완결

2. 나의 독서 상태
   - `reading`: 읽는 중
   - `finished`: 완주
   - `dropped`: 중도 포기

독서 상태는 분류와 필터링에만 사용한다. 개인 리뷰, 완주 후기, 중도 포기 사유는 저장하지 않는다.

## 업데이트 모델

- `episodeCount`: 공식/공개 기준 현재 회차 수다.
- `updateWeekdays`: 공식/공개 기준 업데이트 요일 배열이다. 완결 작품은 빈 배열이다.
- `updateScheduleLabel`: 화면에 보여줄 업데이트 요일 라벨이다.
- `updateScheduleSource`: 요일 정보 출처다. 공식 출처가 없으면 최신 회차 날짜 기준 추정 여부를 남긴다.
- `latestEpisodeUpdatedAt`: 최신 회차 공개일이다. 상세 패널에서 오늘 기준 경과일 계산에 사용한다.
- `officialUrl`: 공식 플랫폼 작품 URL이다. 상세 패널의 `보러가기` 링크에 사용한다.

## 평점 모델

- `userRating`: 사용자가 직접 기록한 개인 평점이다. 데이터에는 저장할 수 있지만 카드, 목록, 상세 패널에는 표시하지 않는다.
- 독자 점수는 앱 사용자가 상세 패널에서 1점부터 5점까지 줄 수 있다.
- 독자 점수는 `localStorage`에 작품 ID별로 하나만 저장한다. 같은 브라우저에서는 중복 평가가 아니라 기존 평가 수정으로 처리한다.
- 정적 GitHub Pages 배포만으로는 모든 방문자의 독자 점수를 서버에서 합산할 수 없다. 전역 집계가 필요하면 별도 백엔드나 저장소를 붙인다.

## 구현 범위

- TypeScript 프론트엔드 앱을 구성한다.
- 데이터베이스는 사용하지 않고 `webtoons/webtoons.json`을 데이터 소스로 사용한다.
- `webtoons/webtoons.xml`은 보존용 원본으로 두고, 앱은 변환이 쉬운 보조 JSON을 읽는다.
- 작품 표지는 `webtoons/covers`에 저장하고 Vite의 정적 asset으로 로드한다.
- 작품별 소개 파일은 `webtoons/description`에 저장해 데이터 검증 대상에 포함한다.
- 입력 컨벤션은 별도 문서에 정리한다.
- 화면은 웹툰 사이트의 목록 탐색 느낌을 참고하되, 개인 기록용 서비스로 표현한다.

## 후속 계획: 모바일 대기열 입력 앱

핸드폰에서 새 웹툰을 빠르게 기록하기 위한 별도 입력 플로우를 추가한다.

- 사용자는 제목, 별점, 독서 상태, 읽은 화수만 입력한다.
- 앱은 GitHub REST API로 `queue/pending-webtoons.jsonl`에 JSON Lines 한 줄을 append한다.
- 입력 앱은 정식 아카이브 파일을 수정하지 않는다.
- 나중에 Codex가 대기열을 읽고 공식 표지, 작가, 장르, 소개글을 찾아 정식 아카이브에 반영한다.
- 구현 순서는 `docs/mobile-queue-app-plan.md`의 WTQ 티켓을 따른다.

## 제외 범위

- 외부 웹툰 사이트 크롤링
- 불법 콘텐츠 링크 제공
- 사용자 인증
- 서버 DB 저장
- 실제 이미지 업로드 시스템
- 여러 방문자의 독자 평점을 서버에서 합산하는 기능
- 모바일 대기열 입력 앱에서 공식 웹툰 정보를 자동 보강하는 기능

## 완료 기준

- 문서에 정의된 상태값과 JSON 컨벤션이 구현에 반영되어 있다.
- `연재중`/`완결` 탭으로 목록이 필터링된다.
- 감상 상태 탭으로 `읽는 중`, `완주`, `중도 포기` 기록이 필터링된다.
- 카드와 목록 보기에는 실제 표지, 제목, 작가, 플랫폼, 장르, 진행도, 작품 소개가 표시된다.
- 상세 패널에는 공식 사이트 `보러가기`, 작품 소개, 업데이트 요일, 최신 회차 공개일, 오늘 기준 경과일, 독자 평점 입력, 보정 note, 장르 전체가 표시된다.
- 통계 대시보드는 현재 아카이브 기준으로 그래프를 표시한다.
- 웹툰 추가는 `webtoons/webtoons.json`, `webtoons/webtoons.xml`, `webtoons/covers`, `webtoons/description` 갱신으로 가능하다.
- 아카이브 변경 뒤 `npm run validate:webtoons`로 XML, JSON, 표지, 설명 파일 매칭을 검증한다.
