# 웹툰 기록 페이지 작업 정의

## 목표

개인 웹툰 감상 기록을 정리하는 TypeScript 기반 웹 페이지를 만든다. 불법 공유나 외부 감상 기능이 아니라, 사용자가 이미 읽었거나 읽는 중인 웹툰을 개인적으로 분류하고 리뷰를 남기는 기록용 페이지다.

## 핵심 화면

참고 이미지처럼 어두운 배경의 웹툰 목록형 UI를 만든다.

- 상단에는 큰 상태 탭을 둔다.
  - `연재`
  - `완결`
- 그 아래에는 독서 상태 세부 탭을 둔다.
  - `완주`
  - `중도 포기`
- 웹툰 목록은 카드 그리드로 보여준다.
- 카드에는 표지, 플랫폼, 장르, 제목, 작가, 진행 상태, 별점, 리뷰 또는 중도 포기 사유 요약을 보여준다.
- 검색, 플랫폼 필터, 장르 필터, 정렬 옵션을 제공해 기록이 많아져도 찾기 쉽게 한다.

## 상태 모델

웹툰 하나는 두 가지 축으로 분류한다.

1. 작품 연재 상태
   - `ongoing`: 연재
   - `completed`: 완결

2. 나의 독서 상태
   - `finished`: 완주
   - `dropped`: 중도 포기

`finished`인 웹툰은 리뷰를 남긴다.  
`dropped`인 웹툰은 중도 포기 사유를 남긴다.

## 구현 범위

- TypeScript 프론트엔드 앱을 구성한다.
- 데이터베이스는 사용하지 않고 `webtoons/webtoons.json`을 데이터 소스로 사용한다.
- `webtoons/webtoons.xml`은 보존용 원본으로 두고, 앱은 변환이 쉬운 보조 JSON을 읽는다.
- 입력 컨벤션은 별도 문서에 정리한다.
- 화면은 웹툰 사이트의 목록 탐색 느낌을 참고하되, 개인 기록용 서비스로 표현한다.

## 제외 범위

- 외부 웹툰 사이트 크롤링
- 불법 콘텐츠 링크 제공
- 사용자 인증
- 서버 DB 저장
- 실제 이미지 업로드 시스템

## 완료 기준

- 문서에 정의된 상태값과 JSON 컨벤션이 구현에 반영되어 있다.
- `연재중`/`완결` 탭으로 목록이 필터링된다.
- 감상 상태 탭으로 `읽는 중`, `완주`, `중도 포기` 기록이 필터링된다.
- 카드에는 실제 표지, 제목, 작가, 플랫폼, 장르, 진행도, 작품 소개가 표시된다.
- 상세 패널에는 작품 소개, 보정 note, 장르 전체가 표시된다.
- 웹툰 추가는 `webtoons/webtoons.json`, `webtoons/webtoons.xml`, `webtoons/covers`, `webtoons/description` 갱신으로 가능하다.
